**** DONE Cross-browser + mobile pass (watch the Safari "skips around" issue)
CLOSED: [2026-05-30]
Confirmed working on-device (desktop Safari + iOS) after the fix below.
Root cause of the Safari "skips around" found and fixed: simple.css sets
`html { scroll-behavior: smooth }` globally, so even the engine's direct
scrollLeft/scrollTop assignments animated, and each per-move recenter
interrupted the previous one — the fighting animations are the "skipping".
Fixes:
- public/css/game.css: `.viewport { scroll-behavior: auto }` (opt the map out of
  the site-wide smooth scroll) + `-webkit-overflow-scrolling: touch`.
- public/js/engine.js: centerCamera now drives the scroll with its own
  cancelable requestAnimationFrame tween (easeOutQuad, 160ms) and cancels any
  in-flight tween before starting a new one, so rapid moves can't stack.
Verified headlessly (no JS errors, centering math correct). STILL NEEDS a human
on-device confirmation in desktop Safari + iOS Safari (camera follow on the new
bridges + the warp transition).
